  • createFetchTestResult(timingEntry: ResourceTimingEntry, response: Response, testConfig: unknown, setupResult: TestSetupResult): Promise<TestResultBundle>
  • remarks

    A provider implements this in order to define its logic for creating a {@link ResultBundle} describing the outcome of running a Fetch test.

    Parameters

    • timingEntry: ResourceTimingEntry

      The Resource Timing entry used to generate the test result.

    • response: Response

      The Response object resulting from the fetch activity.

    • testConfig: unknown

      The test configuration.

    • setupResult: TestSetupResult

      The provider-defined result of any test setup activity done prior to the fetch.

    Returns Promise<TestResultBundle>
